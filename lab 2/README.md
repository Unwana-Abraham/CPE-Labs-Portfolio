# Lab 2: Advanced HTML & Semantic Structure

## Project Overview
In this lab, I transitioned from basic HTML to **Semantic HTML5**. The goal was to build a structured Student Profile page that is accessible, organized, and validated according to industry standards.

## Key Features Added
* **Semantic Tags:** Used `<nav>`, `<article>`, `<section>`, and `<footer>` to create a meaningful page hierarchy.
* **Data Representation:** Created an expanded HTML `<table>` to manage student records.
* **User Input:** Built a functional `<form>` featuring text inputs, radio buttons for gender selection, checkboxes, and a textarea for comments.
* **Navigation:** Implemented internal "jump links" using **Section IDs** to allow quick navigation between the Profile and Registration areas.

## Technical Reflection
The most challenging part of this lab was understanding the relationship between **Labels and Inputs** (using `for` and `id`). I learned that validating code is essential for catching "nesting" errors that aren't always visible in the browser but affect how the code is read by machines.
